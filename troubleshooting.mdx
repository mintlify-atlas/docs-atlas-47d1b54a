## Video playback

<AccordionGroup>
  <Accordion title="Video won't play">
    The selected provider may be temporarily unavailable or may not carry this episode.

    1. On the episode page, open the provider selector and choose a different provider.
    2. Try a different episode to determine whether the problem is provider-wide or episode-specific.
    3. Check your internet connection — open a browser and confirm you can reach external sites.
    4. Wait a few minutes and try again. Providers can have brief outages.
  </Accordion>

  <Accordion title="Episodes don't load">
    If the episode list fails to appear:

    1. Verify your internet connection.
    2. Pull down to refresh the episode page.
    3. Switch to a different provider — the current provider may be down.
    4. If the problem persists, wait and try again later.
  </Accordion>
</AccordionGroup>

## Installation

<AccordionGroup>
  <Accordion title="Can't install the APK on Android">
    Android blocks installation of apps from outside the Play Store by default.

    1. Go to **Settings** on your Android device.
    2. Search for **Install unknown apps** or navigate to **Apps > Special app access > Install unknown apps**.
    3. Select the app you are using to open the APK (such as your file manager or browser) and enable **Allow from this source**.
    4. Open the APK file again and proceed with the installation.
  </Accordion>

  <Accordion title="App crashes on startup (Android)">
    1. Confirm your device runs Android 5.0 (API 21) or later. Earlier versions are not supported.
    2. Uninstall Anikin and reinstall the latest APK from [GitHub Releases](https://github.com/jerry08/Anikin/releases).
    3. If the crash continues, enable **Developer mode** in **Profile > Settings** to reveal detailed error messages, then report the error on GitHub Issues.
  </Accordion>
</AccordionGroup>

## Account & sync

<AccordionGroup>
  <Accordion title="AniList login not working">
    1. Make sure you have an active internet connection.
    2. On **Android**, the AniList OAuth flow opens inside the app. On **Windows**, it opens in your default web browser — complete the login there and return to Anikin.
    3. If the browser opens but nothing happens after you authorise, close and reopen Anikin, then try logging in again.
    4. If the issue persists, report it on [GitHub Issues](https://github.com/jerry08/Anikin/issues).
  </Accordion>
</AccordionGroup>

## Downloads

<AccordionGroup>
  <Accordion title="Download stuck or failing">
    1. **Check storage permissions (Android)** — go to **Settings > Apps > Anikin > Permissions** and ensure storage access is granted.
    2. **Check available disk space** — downloads fail silently when there is no room left on the device.
    3. **Reduce the parallel download limit** — open **Settings** and lower the **Parallel limit** value (default is 5). High parallel counts can cause failures on slower connections.
    4. Cancel the stuck download and try enqueueing it again.
  </Accordion>
</AccordionGroup>

## Updates

<AccordionGroup>
  <Accordion title="App not updating automatically">
    Anikin uses the GitHub Releases API to check for updates on startup. If the update prompt never appears:

    1. Open **Settings** and confirm **Always check for updates** is enabled.
    2. Check your internet connection.
    3. Update manually by downloading the latest release from [GitHub Releases](https://github.com/jerry08/Anikin/releases).
  </Accordion>
</AccordionGroup>

## Developer mode

<AccordionGroup>
  <Accordion title="Enabling developer mode for detailed error messages">
    Developer mode causes Anikin to display full error details (stack traces and exception messages) in alert dialogs instead of silently failing.

    To enable it:

    1. Open the **Profile** tab.
    2. Go to **Settings**.
    3. Enable **Developer mode**.

    <Warning>Developer mode is intended for debugging. Disable it when you are done to avoid unexpected alert dialogs during normal use.</Warning>
  </Accordion>
</AccordionGroup>
